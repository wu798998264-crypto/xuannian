# 全盘查找媒体筛选与缩略图设计

## 目标

在不阻塞文件索引查询、不降低持续滚动流畅度的前提下，为全盘查找增加文档、图片、视频、音频筛选，并为图片和视频结果显示缩略图。

## 数据流

文件查询仍由 Everything 或 Spotlight 只返回路径、名称、大小和修改时间。文件类型筛选直接下推到索引层；查询完成后界面立即显示结果和文件类型占位图标。缩略图通过独立 IPC 请求系统缩略图服务，只处理虚拟列表当前可见的图片和视频。

## 性能边界

- 虚拟列表最多保留 48 个结果行。
- 缩略图并发固定为 3，缓存最多保留 192 项。
- 新搜索会清空尚未执行的缩略图队列，旧请求不得回写新结果。
- 缩略图尺寸限制在结果行需要的低分辨率范围，视频不由渲染进程自行解码。
- 文档和音频仅显示类型图标，不读取文件内容。

## 交互

筛选栏增加“文档”“图片”“视频”“音频”。图片和视频先显示类型图标，系统缩略图准备好后原位替换，不改变结果行尺寸。视频缩略图带播放标记。截图快捷键和全盘查找快捷键旁增加感叹号按钮，点击后显示用途、默认组合键和冲突说明。

## 验证

单元测试覆盖文档与媒体扩展名和筛选参数；Electron 运行时测试覆盖 1 万条结果下的 DOM 上限、缩略图并发/缓存上限和真实本地图片缩略图；桌面及窄窗口截图检查筛选栏、结果行和设置说明按钮无重叠。
